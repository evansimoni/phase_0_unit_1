What is Pair Programming?

Pair programming is an agile software development technique in which two programmers work together at one workstation. One, the driver, writes code while the other, the observer, pointer or navigator, reviews each line of code as it is typed in. The two programmers switch roles frequently.

Reflection

I came into this pairing session without expectations because I wanted to be forced to think on my feet. Suprisingly, the challenge that Clayton and I worked on was pretty straightforward. I think this one was so easy because it is meant to be a test of our pair programming and communication skills rather than a test of our technical knowledge. Nevertheless, we did encounter a few technical concepts that we made progress on including HTML5 name-specific tags, float positioning and indentation conventions. I think CLayton and I worked effectively and at a good pace. We finished 95% of the exercise within the hour and we each did about 50% of the work.
